# Cost Analysis: AI Input Vernacular Study

## Output / Processing Tokens: The Key Question

**Output tokens are identical across all levels** — always 1 token (a single letter: A/B/C/D).
The model does not process abbreviated inputs more intensively. Input token savings are not eaten by output.

Token savings vs L1:
- L2: **−2.5%** (370 tokens / 114q), same accuracy
- L3: **−8.3%** (1,202 tokens / 114q), −1.7pp accuracy

Savings are identical across tiers in percentage terms; dollar value scales with model price.

## The Scaffolding Problem: Realistic API Cost

The 8.3% savings applies only to question tokens. Every evaluation call ran through Claude Code's
agent infrastructure, adding ~13,600 tokens of fixed scaffolding per call. Question tokens are <1%
of actual billed input:

| Level | Question tokens | Scaffolding | Total/call | Q% of billed | Sonnet cost (114q) |
|---|---|---|---|---|---|
| L1 | 127.6 | 13,600 | 13,727.6 | 0.93% | $4.70 |
| L2 | 124.4 | 13,600 | 13,724.4 | 0.91% | $4.70 |
| L3 | 117.1 | 13,600 | 13,717.1 | 0.85% | $4.69 |

**Realistic L3 savings vs L1 (Sonnet): $0.0036 — 0.08% of actual billed cost.**

For direct API calls (no agent overhead), savings would be the full 8.3%.

## What Failed at L3: Subject Analysis

Only 3 of 57 subjects degraded; all were 100%→50% (1 wrong out of 2). All 3 held at L2,
meaning the failure is specifically caused by **word truncation** (the L3-specific step):

| Subject | L1 | L2 | L3 | Pattern |
|---|---|---|---|---|
| clinical_knowledge | 100% | 100% | 50% | Medical terminology mangled by truncation |
| marketing | 100% | 100% | 50% | Business terms become ambiguous |
| moral_scenarios | 100% | 100% | 50% | Abstract ethical vocabulary disrupted |

- **53/57 subjects**: no degradation at L3
- **1/57 subjects**: improved (high_school_geography: 50%→100%)
- All 3 failures are domain-specific vocabulary that becomes ambiguous when truncated

## Summary

| Question | Answer |
|---|---|
| Do output tokens differ across levels? | **No** — always 1 token (single letter) |
| Does abbreviation cause more model processing? | **No** — output is identical |
| L3 token savings vs L1? | 8.3% of question text; 0.08% of billed input (with scaffolding) |
| L3 dollar savings (Sonnet, 114q)? | $0.0036 question-only / ~$0.000034 realistic |
| L3 total cost with Haiku abbreviation? | $1.05 vs $0.05 for L1 — 22× more expensive |
| Break-even for L3 (Sonnet, realistic)? | ~30,000 evaluation runs on the same dataset |
| Which subjects fail at L3? | clinical_knowledge, marketing, moral_scenarios |

## Conclusion

**L2 is the practical winner**: rule-based texting abbreviations are free to generate, preserve full
accuracy (95.6%), and save 2.5% of question-level tokens. L3's word truncation saves more tokens
(8.3%) but costs $1.01 to generate via Haiku, reduces accuracy slightly (−1.7pp), and never
recoups its generation cost at typical dataset sizes and evaluation frequencies.

For direct API deployments without agent scaffolding, L3 becomes break-even after ~280 re-uses of
the same dataset. With agent/Claude Code overhead, break-even is ~30,000 runs — effectively never.
