## Key Design Decisions

### Token Estimation
- Input tokens estimated via `chars ÷ 4` (Claude's avg tokenization rate)
- Output tokens hardcoded to 1 (single letter answer)
- These are **estimates**, not actual API usage
- Sufficient for relative comparisons between L1/L2/L3

### Error Handling
- Both workflows use `.catch()` handlers on agent promises
- Failed agent calls return `null` and are filtered out
- Warnings logged for failed calls
- Length assertions prevent silent data loss

### Shell Injection Prevention
- Both workflows use heredoc pattern for large JSON payloads
- Prevents single quotes in MMLU questions from breaking shell commands
- Example: "Newton's law" won't cause shell injection

### Concurrency
- Phase 1: 114 parallel agents (L3 generation)
- Phase 2: 342 parallel agents (evaluation)
- Claude Code caps concurrency at ~16 simultaneous agents
- Total runtime dominated by longest single call, not sum of all calls
